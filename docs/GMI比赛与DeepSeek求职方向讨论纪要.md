### 三条硬约束（多轮迭代沉淀出来的"筛选标准"）
1. **"出海产品" = 做面向海外用户的产品**（终端用户在海外、英文优先），**不是**服务出海这件事、**更不是**跨境电商。

## 四、选题演化过程（被否决的方向 + 原因，重要记录）

| # | 方向 | 否决原因 |
|---|---|---|
| 1 | 出海市场情报 / 调研 Agent | 是"服务出海这件事"的工具，不是出海**产品** |
| 2 | Coding Agent（自纠错修 bug） | 误读赛道；本人坚决不做 coding agent；市场饱和（OpenCode 172k★/Claude Code/Cursor） |
| 3 | TruthLens（多模态通用事实核查） | 太复杂、像"能力"不像产品；名字不亮；**核查对象不明确、需求量存疑** |
| 4 | ScamLens（跨境网购防骗/防假货） | 又滑回**服务跨境电商**；辨假货/识骗局需电商行业 know-how，本人没有 |
| 5 | HypeLens（AI 吹牛/跑分核查） | **没钉死核查对象**、需求量不大，定位不够清晰、不够"产品" |
| 6 | PrintMedic（3D 打印翻车诊断） | 场景钉死、需求浓、蹭打印机奖项，但**3D 打印需查陌生知识库 → 缺 know-how** |

### 沉淀出的关键认知
- **诚实口径教训**：自动事实核查科学上不可靠（GPT-4o 拒答 43%、整体准确率 ~0.75）。**严禁宣称"确保都是真新闻/保真"**；改为"评分 + 证据 + 置信度 + 证据不足显式标存疑"，正好回应复盘里必被问的 RAG 严谨性。
- **市场调研结论**：① "多模态深度研究"是 2026 公认空白区（头部 deep research 全文本，多模态仍停在 arXiv：MM-DeepResearch/WebWatcher/MIRAGE）；② "拍照→诊断→修"已有玩家（iFixit FixBot、Jenova 汽修、现场运维助手）→ 通用维修被占，**垂直 + 真 agentic 仍有空当**；③ 视觉 Agent 底座成熟（Project Astra、AskUI、Qwen3-VL、GLM-4.6V）。

## 五、最终定稿方向（2026-06-29 锁定）

### ✅ OfferCheck —— 求职/机会防坑研究 Agent

**场景钉死**：求职者答应一份工作 / offer 之前 → 丢入 JD、招聘方聊天截图或公司名 → Agent **自主联网调查 + 主动证伪**（识别假招聘/骗个人信息/骗入职押金/骗 take-home 代码等套路；查公司是否真实存在、招聘方身份是否对得上、域名邮箱是否山寨、有无受害者爆料）→ 输出 **「靠谱 / 存疑 / 大概率有坑」+ 红旗证据链 + 你还需亲自确认的 N 件事**，**不是一篇报告**。人设皮 = "怀疑型研究管家"。

**为什么是它（过全部约束）**：真 agent（开放式调查，路径随发现变、撞死胡同绕路 Reflexion）✅；证伪为核（productize Nexa 的 Verifier + 多源交叉验证）✅；非报告（给裁定+坑）✅；场景钉死（对象=一个求职机会）✅；真刚需且高频（全球远程/海外求职假招聘泛滥）✅；价值=调查+推理，无陌生 KB ✅；**本人正求职=用户本人、零 know-how 缺口** ✅；C 端不服务市场、出海仅作英文外壳 ✅；demo 炸（贴可疑远程 offer→看它扒出骗局）✅；对 DeepSeek = "多工具自主调查 Agent + Eval + Trace" 叙事 ✅。

**名字**：OfferCheck / RealOffer / 防坑管家。**roadmap（仅愿景页）**：同一管家扩到投资 / 租房 / 合作等"机会防坑"，比赛只做求职这一刀。

**另一被否方向（记录）**：虚拟陪伴管家（游走微信替你回朋友）——否决原因：① 微信/WhatsApp 明令禁止自动化、封号风险；② 陪伴类红海（128+ App）+ 监管雷区（GUARD Act、Replika GDPR €5M）；③ 几乎用不上 Nexa、对 DeepSeek 不加分。仅保留其"人设皮 + 证伪魂"思路融入 OfferCheck。

### （历史备选）ClaimSnap —— 破损快递 / 网购 → 一键理赔 Agent
- **场景钉死**：网购收到**破损/错发** → **拍张照** → Agent 评估损坏程度、生成带证据的退款/理赔说明，可直接发给商家/平台。
- **过约束自检**：
  - 场景明确 ✅（对象=破损商品照片；用户=被坑的网购者；输出=定损 + 证据 + 理赔信）
  - 真实刚需且高频 ✅（全球网购破损/错发维权）
  - **价值来自本人 know-how**：视觉定损（缺陷/损坏分级，本人目标检测/巡检本行）+ 物流破损/理赔逻辑（京东本行）—— **不靠陌生 KB** ✅
  - C 端自助维权产品，**不服务电商市场** ✅
  - demo 炸 ✅；仍是多工具 Agent（视觉定损 + 查退货政策 + 起草 + 记忆）→ DeepSeek 叙事不丢 ✅
- **名字候选**：`ClaimSnap` / `DamageProof` / `DOA`（Damaged-On-Arrival）。

### 备选：寄件存证 Agent
- 寄贵重物 / 二手平台 / 搬家前，拍照生成"带时间戳的出库状况证据"，到货争议时作铁证。同为物流 + 视觉本行，场景钉死，需求面较窄。

## 六、统御原则与执行计划

### 原则：一台引擎，主线供 DeepSeek，顺手挤出比赛副产品
1. 比赛专属活（薄壳 UI + 录视频）**≤ 2 天**，其余投在引擎 + 八股 + 投递。
2. **7/6 提交后比赛清零**（除非进 TOP5，7/10 揭晓）。
3. **全程用 Claude Code/Cursor 开发** → 既是 DeepSeek JD 第一硬要求"重度 Agent 工具用户"的素材，又把比赛 demo 写了。
4. **接 GMI 仅 API 切换**（base_url/model/key），顺手做，不占主线。

### 比赛产物要扩充的能力（工程方向视角，按 ROI）
- **必做**：① 多模态作为第一入口；② Eval Harness（量化准确率/拒答率/失败模式 + 回归门禁）；③ Trace 可观测（工具/信源/置信度/耗时/成本）。
- **有时间再做**：MCP（把工具包成 server）；Reflexion + Verifier 已有，demo 里显式秀出。
- **不做**：情报采集管道、持续监控、预测功能、重记忆 —— 堆的是 DeepSeek 不奖励的产品广度。
